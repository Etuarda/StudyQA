# Glossário de QA

Este glossário reúne termos importantes de QA e testes de software para consulta rápida durante os estudos.

## A

### Aceitação

Validação final para verificar se o software atende às expectativas do usuário, do cliente ou do negócio antes da entrega.

### Ambiente de teste

Local configurado para executar testes, com sistema, dados, ferramentas, integrações e configurações necessárias.

### API

Interface que permite a comunicação entre sistemas, aplicações ou serviços.

### Automação de testes

Uso de scripts e ferramentas para executar verificações automaticamente, reduzindo esforço repetitivo e aumentando a velocidade de feedback.

## B

### Bug

Nome comum para defeito encontrado em um software, documento, requisito ou outro artefato.

## C

### Caixa branca

Técnica de teste em que a estrutura interna do código é conhecida e usada para criar ou executar os testes.

### Caixa preta

Técnica de teste em que o sistema é avaliado por entradas, saídas e comportamento externo, sem analisar sua implementação interna.

### Caso de teste

Conjunto de passos, dados, condições e resultado esperado usado para validar um comportamento específico do sistema.

### Causa raiz

Origem real de um problema. Identificar a causa raiz ajuda a evitar que o mesmo tipo de defeito se repita.

### Cenário de teste

Situação ou fluxo que deve ser validado. Normalmente é mais amplo que um caso de teste.

### Cobertura de teste

Medida do quanto uma parte do sistema, requisito, código ou fluxo foi exercitada pelos testes.

## D

### Defeito

Problema inserido em um artefato, como código, documentação, requisito ou caso de teste, geralmente causado por um erro humano.

## E

### End-to-end

Teste de ponta a ponta que simula um fluxo completo do usuário, da entrada até a saída esperada.

### Erro

Engano humano que pode gerar um defeito em algum artefato do software.

## F

### Falha

Comportamento incorreto observado quando um defeito é ativado durante a execução do software.

## I

### Integração

Processo de conectar componentes, módulos, sistemas ou serviços para que funcionem em conjunto.

## M

### Manutenção

Alteração feita em um software já existente, como correção de falhas, melhoria ou criação de nova versão.

## Q

### QA

Quality Assurance, ou Garantia da Qualidade. Área responsável por apoiar processos, práticas e testes que aumentam a qualidade do produto.

### Qualidade de software

Grau em que um software atende aos requisitos, expectativas dos usuários, necessidades do negócio e atributos como segurança, desempenho, usabilidade e manutenção.

## R

### Regressão

Teste feito após mudanças no sistema para verificar se funcionalidades existentes continuam funcionando corretamente.

### Requisito funcional

Descrição do que o sistema deve fazer, como regras, fluxos e funcionalidades.

### Requisito não funcional

Descrição de como o sistema deve se comportar, considerando desempenho, segurança, usabilidade, confiabilidade e outros atributos de qualidade.

### Regra de 10 de Myers

Ideia de que o custo para corrigir um defeito tende a aumentar quando ele é encontrado mais tarde no ciclo de desenvolvimento, podendo crescer cerca de 10 vezes a cada etapa.

## T

### Teste de aceitação

Teste que valida se o sistema atende às necessidades e expectativas dos usuários, cliente ou negócio.

### Teste de componente

Teste de uma parte isolada do código, como função, módulo ou classe. Também é chamado de teste unitário.

### Teste de confirmação

Teste executado depois da correção de um bug para confirmar se o defeito foi realmente resolvido.

### Teste de integração

Teste que verifica se componentes, módulos, APIs, bancos de dados ou sistemas se comunicam corretamente.

### Teste de manutenção

Teste realizado em um sistema já em produção ou em evolução, após correções, melhorias ou mudanças.

### Teste de regressão

Teste que verifica se uma alteração não quebrou funcionalidades que já existiam.

### Teste de sistema

Teste do software como um todo, validando requisitos funcionais e não funcionais.

### Teste dinâmico

Teste que executa o software para observar seu comportamento.

### Teste estático

Análise de código, documentos ou artefatos sem executar o software.

### Teste funcional

Teste que verifica se uma funcionalidade atende aos requisitos, regras de negócio e comportamento esperado.

### Teste não funcional

Teste que avalia como o sistema se comporta em atributos como desempenho, segurança, usabilidade, compatibilidade e confiabilidade.

### Teste unitário

Teste de uma unidade pequena e isolada do código. Normalmente é criado e executado por desenvolvedores.

## U

### UAT

User Acceptance Testing, ou Teste de Aceitação do Usuário. É realizado por usuários finais ou representantes do negócio para validar se o sistema atende às necessidades reais.
